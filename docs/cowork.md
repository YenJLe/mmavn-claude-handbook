# Claude Cowork

*Cowork is the second way to work with Claude. Where the desktop app is a chat window, Cowork is a folder on your computer that Claude reads, writes to, and reasons across.*

!!! note "Full walkthrough coming in session 2"
    This page is a primer so you know what the term means and when to reach for each mode. The live install and hands-on setup happen in the second training session.

## What Cowork is

Cowork is Claude running against a folder on your computer. You point it at a folder and Claude can read any file inside it, edit files in place, create new ones, and maintain a persistent sense of the project across sessions. The folder becomes the workspace.

Where the desktop app is a conversation, Cowork is a collaborator who lives in your files.

## Chat vs Cowork: the core difference

|                                | **Chat (desktop app)**                                 | **Cowork (folder-based)**                                    |
|--------------------------------|--------------------------------------------------------|--------------------------------------------------------------|
| **Where you work**             | A chat window                                          | A folder on your computer                                    |
| **Where files live**           | Uploaded into a Project                                | On your disk, inside the folder                              |
| **What Claude does**           | Reads what you paste or upload and replies             | Reads, writes, and edits files directly                      |
| **How you get output**         | Copy from the chat or artifact panel                   | Claude updates files in place; you see the changes           |
| **Best for**                   | Drafting, brainstorming, one-off tasks                 | Ongoing projects, multi-file work, structured documentation  |
| **Context between sessions**   | Project instructions + Project knowledge               | Files in the folder (CLAUDE.md, DIARY.md, PROJECT_STATUS.md) |

## Projects in Chat vs Projects in Cowork

Both are called "Projects," and both exist to give Claude persistent context. They are structurally different.

### Projects in Chat

- **Instructions** live in a single text box on the Project page.
- **Knowledge** is a set of files uploaded into the Project.
- Claude reads them at the start of every new chat; it cannot edit them.
- You produce drafts in chat, then copy them out manually.
- Best when the output is content: an email, a deck, a LinkedIn post, a summary.

### Projects in Cowork

- **Instructions** live in a `CLAUDE.md` file inside the folder. You update them by editing the file.
- **Knowledge** is whatever files sit in the folder: briefs, source docs, past decks, decision logs (`DIARY.md`), status dashboards (`PROJECT_STATUS.md`), and anything else.
- Claude reads them at the start of every session and can edit them as work happens.
- The output is the folder itself, updated in place, not a chat transcript.
- Best when the output is a workspace: a plan that evolves over weeks, a deck you iterate on across several sessions, an ongoing research file.

The five foundational files generated by the [project-setup skill](desktop-tour.md#skip-the-blank-page-the-project-setup-skill) work for both. Drop them into a Chat Project's instructions and knowledge, or put them in a Cowork folder as the starting state. Same files, different modes.

## When to reach for each

- **Chat** when the task is a thing you will draft, approve, and send. Invites, articles, decks, summaries, research answers, translations.
- **Cowork** when the task is a workstream with many files you will touch repeatedly. A three-month event, a content library, a strategy doc that evolves, an operating playbook.

Most of the team will live in Chat most of the time and reach for Cowork when a project outgrows a single chat.
